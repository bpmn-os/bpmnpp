80 /**
81  * @brief Represents a BPMN model with all its processes and message flows.
82  *
83  * The Model class reads a BPMN model from a file and provides access to all processes
84  * with their content as well as to all message flows in a BPMN model.
85  * @see @ref Process, @ref MessageFlow
86  * @note The BPMN model is expected to conform with the BPMN specification, e.g.,
87  * it is expected that all boundary events and start events of event subprocesses have
88  * an event definition.
89  * @par
90  * @note All nodes in a process model are derived from an abstract class @ref Node. The
91  * class model uses multiple inheritance to provide type specific attributes. References
92  * are usually provided by pointers to a base class. Casting is required to gain access
93  * to type specific attributes.
94  * @par
95  * @note The BPMN extension mechanism can be used by providing a custom model class derived
96  * from @ref Model and a custom extension derived from the abstract base class
97  * @ref ExtensionElements. The following example shows how an object of the custom extension
98  * `MyExtensionElements` class can be bound to processes by overriding the method
99  * @ref Model::createProcess in the custom model `MyModel` class:
100  * ```
101  * std::unique_ptr<BPMN::Process> MyModel::createProcess(XML::bpmn::tProcess* process) {
102  * return bind<BPMN::Process>(
103  * BPMN::Model::createProcess(process),
104  * std::make_unique<MyExtensionElements>(process)
105  * );
106  * }
107  * ```
108  *
109  * @warning Multiple event definitions are not yet supported. A `std::runtime_error`
110  * will be thrown when parsing an event with multiple event definitions.
111  *
112  * References between different classes are automatically determined:
113  * - For each sequence flow and each message flow, pointers to the source and target
114  * are provided.
115  * @see @ref SequenceFlow, @ref MessageFlow, @ref FlowNode, @ref Process
116  *
117  * - For each node, pointers to receiving and sending message flows are provided.
118  * @see Node
119  * - For each BPMN element that may have child nodes within its scope, the child nodes
120  * and sequence flows between them are owned by the node. Pointers to each flow node,
121  * event subprocess, start events, compensation activities, and compensation event
122  * subprocess are given.
123  * @see @ref Scope
124  * - For each node within a scope, a pointer to the parent scope is provided.
125  * @see @ref ChildNode
126  * - For each node that may receive a flow token, pointers to all incoming and outgoing
127  * sequence flows are given.
128  * @see @ref FlowNode, @ref SequenceFlow
129  * - For each activity, pointers to each boundary event (excluding the compensation
130  * boundary event) and to the compensation activity or compensation event subprocess
131  * are provided.
132  * @see @ref Activity, @ref BoundaryEvent
133  * - For each event attached to the boundary of an activity, a pointer to the activity
134  * is provided.
135  * @see @ref Activity, @ref BoundaryEvent
136  * - For each link event, a pointer to the respective target or source(s) is provided.
137  * @note Target and sources are matched based on the
138  * @ref XML::bpmn::tLinkEventDefinition::name attribute in the link event
139  * definition. If no such name is given, the @ref FlowNode::name attribute of the link
140  * event is used as fallback.
141  * @attention For each link source exactly one link target must be found, otherwise a
142  * `std::runtime_error` will be thrown.
143  * @see @ref LinkSourceEvent, @ref LinkTargetEvent, @ref XML::bpmn::tLinkEventDefinition,
144  * @ref FlowNode
145  * - For each throwing compensation event, a pointer to the activity to be compensated is
146  * provided.
147  * @note The respective activity is determined based on the
148  * @ref XML::bpmn::tCompensateEventDefinition::activityRef attribute of the
149  * compensation event definition and the @ref Node::id attribute of the activity. If
150  * no such attribute reference is given, respective activity is determined based on the
151  * @ref FlowNode::name attribute of the compensate throw event and the @ref FlowNode::name
152  * of the activity.
153  * @attention Compensation throw events in compensation event subprocesses can only trigger
154  * compensation of activities within parent scope. All other compensation throw events can
155  * only trigger compensation of activities within the same scope.
156  * @see @ref CompensateThrowEvent, @ref XML::bpmn::tCompensateEventDefinition, @ref Activity,
157  * @ref FlowNode
158  **/
